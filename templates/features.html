Release-cycle scheduling

Plan a 14-day rollout in one operation.

Pick a release date. Cadence builds the rollout calendar across every connected account — staggered cadence, gap rules enforced, peak posting windows respected.

  • Pre-built 7-day, 14-day, 30-day rollout templates
  • Auto-stagger to avoid same-hour collisions
  • Configurable minimum gap between posts per account
  • Custom rollout templates per workspace
Per-brand voice

The label talks like the label.

Each connected account gets its own caption template, hashtag set, and CTA library. The label doesn't sound like the artist. The show doesn't sound like the production company. Voice stays consistent without copy-pasting.

  • Caption templates per connected account
  • Hashtag sets per brand (rotating + locked)
  • CTA library (link in bio, comment-to-buy, etc.)
  • Variable substitution (artist name, release date, etc.)

Built on official TikTok APIs

No browser hacks. No reminders. Just scheduled → published.

Cadence uses TikTok's Content Posting API — the same official integration Buffer, Later, and Hootsuite use. Posts publish directly at the scheduled time. No "open the app to confirm." No browser-automation ban risk.

Direct publish

Video bytes upload to TikTok via the official chunked upload endpoint. At the scheduled time, the post goes live. No human in the loop.

Privacy + safety

OAuth tokens encrypted at rest with AES-256. Per-user isolated storage. Revoke access anytime via TikTok's app permissions page.

Performance pulled back

View counts, likes, comments fetched 1h / 24h / 7d after publish. Drives the post-rollout analytics dashboard.
